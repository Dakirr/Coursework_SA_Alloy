# Формальная реализация системы на языке Alloy

Формальная модель системы на языке Alloy для дальнейшей возможности программно проверять гипотезы о состоянии системы.

## Сигнатуры системы

1. **Number**
   - Служебная сигнатура, представляет собой перечисление N0, N1 ... – номеров объектов

2. **Time** `<t: Int>`
   - Сигнатура, содержащая информацию о временной точке
   - В исходной постановке соответствует $\mathbb{T}$

3. **Person** `<id:Number>`
   - Сигнатура, содержащая информацию об объекте (островитянине)
   - В исходной постановке соответствует $B$

4. **QualityType**
   - Служебная сигнатура, представляет собой перечисление типов событий
   - Содержит два непересекающихся подмножества: **ConstantQualityType** и **ChangingQualityType**
   - В исходной постановке соответствует $E$

5. **Quality** `<p: Person, t: Time, type: QualityType, value: Number>`
   - Сигнатура, отвечающая за «обладание признаком *value* типа *type* человеком *p* в момент времени *t*»
   - В исходной постановке соответствует значению функции $\mathbb{S}$

6. **TravellingEvent** `<p: Person, s: Time, a: Time, from: Number, to: Number>`
   - Сигнатура, отвечающая за «событие отправления человека *p* в путь из дома *from* в момент *s* с прибытием в дом *to* в момент *a*»
   - В исходной постановке соответствует подмножеству Q

7. **MeetingEvent** `<ps: some Person, t: Time, h: Number>`
   - Сигнатура, отвечающая за «событие встречи в доме *h* в момент времени *t* множества людей *ps*»
   - В исходной постановке соответствует подмножеству Q

8. **ExchangeEvent** `<p1: Person, p2: Person, t: Time, type: ChangingQualityType>`
   - Сигнатура, отвечающая за «передачу человеком *p1* своего признака типа *type* игроку *p2* в момент *t*»
   - В исходной постановке соответствует подмножеству Q
